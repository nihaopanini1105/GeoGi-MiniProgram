# GeoGi 客户诊断报告模板

适用位置：微信小程序报告页、飞书报告管理、后续 PDF/网页报告。

## 客户可见结构

1. 报告封面
   - 品牌名称
   - 项目编号
   - 报告状态
   - 更新时间

2. 诊断摘要
   - 综合可见度分数
   - 一句话结论
   - 本次检测范围

3. 核心评分
   - 品牌识别：AI 是否知道品牌是谁、做什么
   - 主动推荐：用户提出真实需求时，AI 是否会推荐品牌
   - 信息准确：AI 对业务、优势、市场和联系方式的描述是否准确
   - 信源可信：AI 回答是否有可追溯、可信赖的公开来源

4. 关键发现
   - 品牌没有被提到
   - 品牌被提到但没有被推荐
   - AI 对品牌业务理解错误或不完整
   - 竞品被优先推荐
   - 回答缺少可信信源

5. 平台表现
   - 平台名称
   - 提问内容
   - 回答摘要
   - 是否提到品牌
   - 是否主动推荐
   - 信息是否准确
   - 竞品压制情况
   - 证据链接或截图

6. 优化建议
   - 品牌实体信息补强
   - 官网和公开页面内容补强
   - 第三方可信信源建设
   - 常见AI问题内容建设
   - 竞品对比内容建设

## 数据来源

- `客户与品牌`：客户提交资料、品牌基础信息、联系方式
- `诊断项目`：项目阶段、负责人、预计交付时间
- `平台测试记录`：平台、问题、回答原文、证据链接
- `回答分析结果`：评分、核心问题、优化建议
- `报告管理`：报告状态、版本、交付说明、下一步建议

## 小程序展示规则

- 报告未生成：显示项目进度和报告模板说明
- 报告初稿已生成：显示评分、平台表现、关键发现和建议
- 部分平台读取受限：展示已读取内容，并提示需要人工复核
- 正式交付前：报告状态仍保留为初稿，避免客户误认为已经最终定稿

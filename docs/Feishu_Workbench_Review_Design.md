# GeoGi 飞书诊断工作台整理标准

## 目标

让人工审核时能快速看清：

- 这条记录属于哪个品牌
- 它在诊断流程里的位置
- 是否需要人工审核
- 应该按什么顺序阅读

## 新增统一字段

所有诊断业务表统一增加：

- `品牌分组`：格式为 `品牌名称｜项目编号`
- `排序键`：格式为 `品牌名称｜项目编号｜信息层级｜序号`
- `信息层级`：例如 `01 官方与主体`、`02 产品与品类`、`03 用户决策`
- `审核状态`：默认 `待人工审核`

建议在飞书每张表中创建一个视图：

- 视图名称：`按品牌审核`
- 分组：`品牌分组`
- 排序：`排序键` 升序
- 筛选：`审核状态` 不等于 `已归档`

这样每个品牌之间会自然分开，同一个品牌下的信息会按诊断流程顺序排列。

## 品牌基础档案

保留重点字段：

- 品牌分组
- 品牌标准名称
- 所属企业
- 行业
- 细分业务
- 品类定位
- 客户提供产品
- 产品归属判断
- 产品核验任务
- 官方渠道
- 公开信源
- 竞品品牌
- 风险备注
- 审核状态

其中 `客户提供产品` 只代表客户提到的具体产品、系列或服务候选，不代表行业通用品类。  
例如：`小黑杯` 是瑞幸咖啡的产品系列候选，行业品类仍应是 `咖啡`。

## 全网信源

程序会生成必要的信源核验任务：

- 官方与主体
- 产品与品类
- 用户决策
- 竞品对比
- 风险核验

每条候选信源会带搜索入口。正式版上线搜索服务后，再把搜索入口替换成自动采集到的真实链接和摘要。

## 数据清理规则

清空测试数据时，只清以下诊断业务表：

- 客户与品牌
- 诊断项目
- 品牌基础档案
- 全网信源
- 品牌关键词
- 行业热门问题
- AI 检测问题
- 平台测试记录
- 回答分析结果
- 报告管理

不清空研究中心文章。
